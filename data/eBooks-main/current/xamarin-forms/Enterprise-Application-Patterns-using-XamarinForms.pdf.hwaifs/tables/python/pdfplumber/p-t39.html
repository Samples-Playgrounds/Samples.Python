0 Tip: Consider caching pages
1 Page caching results in memory consumption for views that are not currently displayed. However,
2 without page caching it does mean that XAML parsing and construction of the page and its view
3 model will occur every time a new page is navigated to, which can have a performance impact for a
4 complex page. For a well-designed page that does not use an excessive number of controls, the
5 performance should be sufficient. However, page caching might help if slow page loading times are
6 encountered.
