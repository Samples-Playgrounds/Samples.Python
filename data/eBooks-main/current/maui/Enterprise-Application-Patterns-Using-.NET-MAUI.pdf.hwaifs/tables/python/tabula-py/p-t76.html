Property Description
0 AllowedGrantTypes Specifies how a client wants to interact with\rIdentityServer. For more information see\rConfiguring the authentication flow.
1 ClientSecrets Specifies the client secret credentials that are\rused when requesting tokens from the token\rendpoint.
2 RedirectUris Specifies the allowed URIs to which to return\rtokens or authorization codes.
3 RequireConsent Specifies whether a consent screen is required.
4 RequirePkce Specifies whether clients using an authorization\rcode must send a proof key.
5 PostLogoutRedirectUris Specifies the allowed URIs to redirect to after\rlogout.
6 AllowedCorsOrigins Specifies the origin of the client so that\rIdentityServer can allow cross-origin calls from\rthe origin.
7 AllowedScopes Specifies the resources the client has access to.\rBy default, a client has no access to any\rresources.
8 AllowOfflineAccess Specifies whether the client can request refresh\rtokens.
9 AllowAccessTokensViaBrowser Specifies whether the client can receive access\rtokens from a browser window.
10 AlwaysIncludeUserClaimsInIdToken Specifies that the user claims will always be\radded to the id token. By default, these would\rhave to be retrieved using the userinfo\rendpoint.
11 AccessTokenLifetime Specifies the lifetime of the access token in\rseconds.
12 IdentityTokenLifetime Specifies the lifetime of the identity token in\rseconds.
