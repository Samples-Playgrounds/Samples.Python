0 Property Description
1 AllowedGrantTypes Specifies how a client wants to interact with\nIdentityServer. For more information see\nConfiguring the authentication flow.
2 ClientSecrets Specifies the client secret credentials that are\nused when requesting tokens from the token\nendpoint.
3 RedirectUris Specifies the allowed URIs to which to return\ntokens or authorization codes.
4 RequireConsent Specifies whether a consent screen is required.
5 RequirePkce Specifies whether clients using an authorization\ncode must send a proof key.
6 PostLogoutRedirectUris Specifies the allowed URIs to redirect to after\nlogout.
7 AllowedCorsOrigins Specifies the origin of the client so that\nIdentityServer can allow cross-origin calls from\nthe origin.
8 AllowedScopes Specifies the resources the client has access to.\nBy default, a client has no access to any\nresources.
9 AllowOfflineAccess Specifies whether the client can request refresh\ntokens.
10 AllowAccessTokensViaBrowser Specifies whether the client can receive access\ntokens from a browser window.
11 AlwaysIncludeUserClaimsInIdToken Specifies that the user claims will always be\nadded to the id token. By default, these would\nhave to be retrieved using the userinfo\nendpoint.
12 AccessTokenLifetime Specifies the lifetime of the access token in\nseconds.
13 IdentityTokenLifetime Specifies the lifetime of the identity token in\nseconds.
