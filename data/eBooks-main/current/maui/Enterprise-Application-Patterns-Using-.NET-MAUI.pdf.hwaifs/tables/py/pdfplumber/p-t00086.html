0 Authentication Flow Description
1 Implicit This flow is optimized for browser-based\napplications and should be used either for user\nauthentication-only, or authentication and\naccess token requests. All tokens are\ntransmitted via the browser, and therefore\nadvanced features like refresh tokens are not\npermitted.
2 Authorization code This flow provides the ability to retrieve tokens\non a back channel, as opposed to the browser\nfront channel, while also supporting client\nauthentication.
3 Hybrid This flow is a combination of the implicit and\nauthorization code grant types. The identity\ntoken is transmitted via the browser channel\nand contains the signed protocol response and\nother artifacts such as the authorization code.\nAfter successfully validating the response, the\nback channel should be used to retrieve the\naccess and refresh token.
