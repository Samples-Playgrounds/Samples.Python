0 Model Characteristics
1 Document Store Data and metadata are stored hierarchically in\nJSON-based documents inside the database.
2 Key Value Store The simplest of the NoSQL databases, data is\nrepresented as a collection of key-value pairs.
3 Wide-Column Store Related data is stored as a set of nested-\nkey/value pairs within a single column.
4 Graph Store Data is stored in a graph structure as node,\nedge, and data properties.
