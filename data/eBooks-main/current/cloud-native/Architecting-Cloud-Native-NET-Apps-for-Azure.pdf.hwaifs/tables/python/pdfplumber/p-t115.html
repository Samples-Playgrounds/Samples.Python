0 Consistency Level Description
1 Eventual No ordering guarantee for reads. Replicas will\neventually converge.
2 Constant Prefix Reads are still eventual, but data is returned in\nthe ordering in which it is written.
3 Session Guarantees you can read any data written\nduring the current session. It is the default\nconsistency level.
4 Bounded Staleness Reads trail writes by interval that you specify.
5 Strong Reads are guaranteed to return most recent\ncommitted version of an item. A client never\nsees an uncommitted or partial read.
