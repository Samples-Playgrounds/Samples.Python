Consistency Level Description
0 Eventual No ordering guarantee for reads. Replicas will\reventually converge.
1 Constant Prefix Reads are still eventual, but data is returned in\rthe ordering in which it is written.
2 Session Guarantees you can read any data written\rduring the current session. It is the default\rconsistency level.
3 Bounded Staleness Reads trail writes by interval that you specify.
4 Strong Reads are guaranteed to return most recent\rcommitted version of an item. A client never\rsees an uncommitted or partial read.
