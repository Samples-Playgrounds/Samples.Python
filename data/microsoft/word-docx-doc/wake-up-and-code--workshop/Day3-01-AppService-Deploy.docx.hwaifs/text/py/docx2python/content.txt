Publish an ASP.NET Core app to Azure with Visual Studio

Source: <a href="https://docs.microsoft.com/en-us/aspnet/core/tutorials/publish-to-azure-webapp-using-vs">https://docs.microsoft.com/en-us/aspnet/core/tutorials/publish-to-azure-webapp-using-vs</a> 



 Important

ASP.NET Core preview releases with Azure App Service

ASP.NET Core preview releases aren't deployed to Azure App Service by default. To host an app that uses an ASP.NET Core preview release, see <a href="https://docs.microsoft.com/en-us/aspnet/core/host-and-deploy/azure-apps/index?view=aspnetcore-2.2#deploy-aspnet-core-preview-release-to-azure-app-service">Deploy ASP.NET Core preview release to Azure App Service</a>.

See <a href="https://blog.xamarin.com/publish-azure-visual-studio-mac/">Publish to Azure from Visual Studio for Mac</a> if you are working on macOS.

To troubleshoot an App Service deployment issue, see <a href="https://docs.microsoft.com/en-us/aspnet/core/host-and-deploy/azure-apps/troubleshoot?view=aspnetcore-2.2">Troubleshoot ASP.NET Core startup errors on Azure App Service</a>.

Set up

--	Open a <a href="https://azure.microsoft.com/free/dotnet/">free Azure account</a> if you don't have one.

Create a web app

In the Visual Studio Start Page, select File > New > Project...

----Image alt text---->File menu<----media/image1.png----

Complete the New Project dialog:

--	In the left pane, select .NET Core.

--	In the center pane, select ASP.NET Core Web Application.

--	Select OK.

----Image alt text---->New Project dialog<----media/image2.png----

In the New ASP.NET Core Web Application dialog:

--	Select Web Application.

--	Select Change Authentication.

----Image alt text---->New Project dialog<----media/image3.png----

The Change Authentication dialog appears.

--	Select Individual User Accounts.

--	Select OK to return to the New ASP.NET Core Web Application, then select OK again.

----Image alt text---->New ASP.NET Core Web authentication dialog<----media/image4.png----

Visual Studio creates the solution.

Run the app

--	Press CTRL+F5 to run the project.

--	Test the About and Contact links.

----Image alt text---->Web application open in Microsoft Edge on localhost<----media/image5.png----

Register a user

--	Select Register and register a new user. You can use a fictitious email address. When you submit, the page displays the following error:

"Internal Server Error: A database operation failed while processing the request. SQL exception: Cannot open the database. Applying existing migrations for Application DB context may resolve this issue."

--	Select Apply Migrations and, once the page updates, refresh the page.

----Image alt text---->Internal Server Error: A database operation failed while processing the request. SQL exception: Cannot open the database. Applying existing migrations for Application DB context may resolve this issue.<----media/image6.png----

The app displays the email used to register the new user and a Log out link.

----Image alt text---->Web application open in Microsoft Edge. The Register link is replaced by the text Hello email!<----media/image7.png----

Deploy the app to Azure

Right-click on the project in Solution Explorer and select Publish....

----Image alt text---->Contextual menu open with Publish link highlighted<----media/image8.png----

In the Publish dialog:

--	Select Microsoft Azure App Service.

--	Select the gear icon and then select Create Profile.

--	Select Create Profile.

----Image alt text---->Publish dialog<----media/image9.png----

Create Azure resources

The Create App Service dialog appears:

--	Enter your subscription.

--	The App Name, Resource Group, and App Service Plan entry fields are populated. You can keep these names or change them.

----Image alt text---->App Service dialog<----media/image10.png----

--	Select the Services tab to create a new database.

--	Select the green + icon to create a new SQL Database

----Image alt text---->New SQL Database<----media/image11.png----

--	Select New... on the Configure SQL Database dialog to create a new database.

----Image alt text---->New SQL Database and server<----media/image12.png----

The Configure SQL Server dialog appears.

--	Enter an administrator user name and password, and then select OK. You can keep the default Server Name.

 Note

"admin" isn't allowed as the administrator user name.

----Image alt text---->Configure SQL Server dialog<----media/image13.png----

--	Select OK.

Visual Studio returns to the Create App Service dialog.

--	Select Create on the Create App Service dialog.

----Image alt text---->Configure SQL Database dialog<----media/image14.png----

Visual Studio creates the Web app and SQL Server on Azure. This step can take a few minutes. For information on the resources created, see <a href="https://docs.microsoft.com/en-us/aspnet/core/tutorials/publish-to-azure-webapp-using-vs?view=aspnetcore-2.2#additonal-resources">Additional resources</a>.

When deployment completes, select Settings:

----Image alt text---->Configure SQL Server dialog<----media/image15.png----

On the Settings page of the Publish dialog:

--	Expand Databases and check Use this connection string at runtime.

--	Expand Entity Framework Migrations and check Apply this migration on publish.

--	Select Save. Visual Studio returns to the Publish dialog.

----Image alt text---->Publish dialog: Settings panel<----media/image16.png----

Click Publish. Visual Studio publishes your app to Azure. When the deployment completes, the app is opened in a browser.

Test your app in Azure

--	Test the About and Contact links

--	Register a new user

----Image alt text---->Web application opened in Microsoft Edge on Azure App Service<----media/image17.png----

Update the app

--	Edit the Pages/About.cshtml Razor page and change its contents. For example, you can modify the paragraph to say "Hello ASP.NET Core!":

HTMLCopy

																@page

																@model AboutModel

																@{

																    ViewData["Title"] = "About";

																}

																<h2>@ViewData["Title"]</h2>

																<h3>@Model.Message</h3>

																

																<p>Hello ASP.NET Core!</p>

--	Right-click on the project and select Publish... again.

----Image alt text---->Contextual menu open with Publish link highlighted<----media/image8.png----

--	After the app is published, verify the changes you made are available on Azure.

----Image alt text---->Verify task is complete<----media/image18.png----

Clean up

When you have finished testing the app, go to the <a href="https://portal.azure.com/">Azure portal</a> and delete the app.

--	Select Resource groups, then select the resource group you created.

----Image alt text---->Azure Portal: Resource Groups in sidebar menu<----media/image19.png----

--	In the Resource groups page, select Delete.

----Image alt text---->Azure Portal: Resource Groups page<----media/image20.png----

--	Enter the name of the resource group and select Delete. Your app and all other resources created in this tutorial are now deleted from Azure.
